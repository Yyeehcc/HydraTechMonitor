# HydroTechMonitor

Веб-портал для мониторинга гидротехнических сооружений (ГТС) Жамбылской области.
Проект показывает, как может выглядеть единая диспетчерская панель для РГП «Казводхоз»:
интерактивная карта объектов, телеметрия с датчиков, оценка состояния и рисков,
диагностика дефектов и формирование технических отчётов.

> **EN.** A monitoring dashboard for hydraulic structures of Zhambyl Region, Kazakhstan.
> Interactive map, simulated IoT telemetry, condition & risk scoring, defect diagnostics and
> printable technical reports. Bilingual interface (RU/EN) with light and dark themes.

---

## О проекте

Жамбылская область — один из самых «водозависимых» регионов Казахстана: сотни каналов,
гидроузлов и водохранилищ, многие из которых построены ещё в советское время. Износ
сооружений и отсутствие наглядного мониторинга усложняют планирование ремонтов.

HydroTechMonitor собирает все объекты на одной карте и даёт оператору быстрый ответ на
вопросы «что в норме», «что требует осмотра» и «что нужно ремонтировать в первую очередь».
Данные датчиков смоделированы, но структура и логика повторяют то, как работала бы
реальная SCADA-панель.

## Возможности

- **Карта объектов** — Leaflet поверх спутниковых тайлов Esri, реальная граница области
  (geoBoundaries) и настоящие русла рек из OpenStreetMap.
- **344 объекта и 6 водохранилищ**, размещённых вдоль действительных рек региона.
- **Телеметрия датчиков** — расход, температура, уровень, мутность, pH и проводимость
  с живым обновлением и мини-графиком за 24 часа.
- **Диагностика** — каталог дефектов ГТС (D01–D13) и неисправностей датчиков (SF01–SF09)
  с рекомендованными мерами по устранению.
- **Фильтры и поиск** — по статусу, району, источнику воды, году постройки, износу и
  активным тревогам.
- **Технический отчёт** — формирование и печать отчёта по объекту на языке интерфейса.
- **Двуязычие и темы** — русский / английский, светлая и тёмная темы.
- **3D-схема** установки датчиков для каждого типа объекта.

## Стек

- HTML, CSS, нативный JavaScript (без сборщиков и фреймворков).
- [Leaflet](https://leafletjs.com/) для карты, Font Awesome для иконок.
- Python (опционально) для генерации и подготовки данных.

Сайт полностью статический — для работы на GitHub Pages бэкенд и Python не требуются.

## Запуск

**Быстро:** откройте `index.html` в браузере.

**С локальным сервером (рекомендуется — корректно грузятся тайлы и данные):**

```bash
python -m http.server 3333
# затем откройте http://localhost:3333
```

## Структура проекта

| Файл / папка | Назначение |
|--------------|------------|
| `index.html` | разметка и точки монтирования интерфейса |
| `style.css` | оформление, темы, анимации, стили печати |
| `app.js` | вся логика: карта, фильтры, панель деталей, отчёты, i18n, 3D |
| `data.js` | сгенерированные объекты и показания датчиков |
| `geo_data.js` | граница области и русла рек (реальные координаты) |
| `i18n_data.js` | англоязычные переводы каталогов (дефекты, неисправности, районы) |
| `canals_data.json` | исходные данные по каналам |
| `gen_data.py` | генератор `data.js` |
| `process_rivers.py` | сборка `geo_data.js` из OSM и geoBoundaries |
| `lib/` | Leaflet и Font Awesome |

## Регенерация данных (опционально)

Нужна только если хотите перестроить датасет. Требуется Python 3.

```bash
# 1) (опц.) пересобрать границу области и русла рек
python process_rivers.py     # -> geo_data.js, rivers_data.json

# 2) перегенерировать объекты и показания датчиков
python gen_data.py           # -> data.js
```

Если правите только интерфейс, эти шаги не нужны — готовые `data.js` и `geo_data.js`
уже лежат в репозитории.

## Планы развития

- Подключение реального источника телеметрии вместо симуляции.
- Экспорт отчёта в PDF на стороне клиента.
- История показаний и тренды за длительный период.
- Адаптация интерфейса под мобильные устройства.

## Источники данных

- Границы региона — [geoBoundaries](https://www.geoboundaries.org/) (ADM1, Kazakhstan).
- Русла рек — [OpenStreetMap](https://www.openstreetmap.org/) через Overpass API.
- Спутниковые тайлы — Esri World Imagery.
